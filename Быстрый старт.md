# Быстрый старт

Инструкция описывает первый запуск проекта в 1C:EDT и настройки, которые нужно выполнить перед началом разработки.

Источники:

1. [1C:Enterprise Development Tools / Установка, обновление / Первый запуск](https://its.1c.ru/db/edtdoc#content:10521:hdoc)
2. [Настройка Git-клиента на Windows](./git-windows.md)

## 1. Установка зависимостей

Для работы с проектом установите:

* Платформу [1С:Предприятие 8.5.1.1302](https://releases.1c.ru/version_files?nick=Platform85&ver=8.5.1.1302). В мастере установки обязательно выберите компонент `Сервер 1С:Предприятия`.
* 1С:EDT [2026.1.0](https://releases.1c.ru/version_files?nick=DevelopmentTools10&ver=2026.1.0). См. [документацию по установке](https://its.1c.ru/db/edtdoc#content:10384:hdoc).
* Git. Для Windows используйте [инструкцию по настройке Git-клиента](./git-windows.md).
* Java Development Kit 17 или выше, если он не был установлен вместе с EDT. Рекомендуется [Axiom JDK 17](https://releases.1c.ru/project/Axiom17FullJDK).

После первой установки Git перезапустите 1C:EDT Start, чтобы обновилась переменная окружения `PATH`.

Проверить установленные зависимости на Windows можно скриптом:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\init_workspace\windows\check-quickstart-deps.ps1
```

Или через `cmd`-обертку:

```cmd
tools\init_workspace\windows\check-quickstart-deps.cmd
```

Для автоматического скачивания и запуска установки платформы и EDT используйте:

```cmd
tools\init_workspace\windows\install-platform.cmd
tools\init_workspace\windows\install-edt.cmd
```

Скрипты скачивают дистрибутивы с `https://releases.1c.ru`, поэтому потребуется логин и пароль пользователя 1С. Подробности запуска описаны в [tools/init_workspace/windows/README.md](../tools/init_workspace/windows/README.md).

## 2. Настройка доступа к репозиторию

1. Настройте SSH-ключ в GitLab: [GitLab SSH Keys](https://gitlab.corp.your.group/-/user_settings/ssh_keys).
1. Клонируйте репозиторий:

```bash
git clone git@gitlab.corp.your.group:mis/itw_mis.git
```

1. Перейдите в каталог проекта:

```bash
cd itw_mis
```

1. Получите актуальное состояние удаленных веток и переключитесь на ветку `dev`:

```bash
git fetch --all --prune
git switch dev
git pull --ff-only
```

## 3. Создание ветки для задачи

Рабочую ветку создавайте от актуальной ветки `dev`.

1. Откройте issue в GitLab и уточните номер задачи.
1. Обновите локальную ветку `dev`:

```bash
git switch dev
git pull --ff-only
```

1. Создайте ветку для задачи:

```bash
git switch -c issue-123-short-description
```

Где `123` - номер issue, а `short-description` - короткое описание латиницей через дефисы.

1. После выполнения задачи зафиксируйте изменения и отправьте ветку:

```bash
git status
git add .
git commit -m "issue-123: краткое описание изменений"
git push -u origin issue-123-short-description
```

1. В GitLab создайте merge request из ветки задачи в `dev`.
1. В описании merge request добавьте ссылку на issue или закрывающую фразу, если процесс проекта это предполагает:

```text
Closes #123
```

## 4. Первый запуск 1C:EDT Start

Запустите `1CEDT Start` одним из способов:

* через ярлык `1CEDT Start` на рабочем столе;
* через меню `Пуск > 1C Enterprise Development Tools > 1CEDT Start`;
* через исполняемый файл `1cedtstart.exe` из каталога установки.

При первом запуске:

1. Нажмите `Создать новый проект`.
1. Авторизуйтесь через портал 1С:ИТС, если 1C:EDT Start попросит выполнить вход.
1. После успешной авторизации вернитесь в 1C:EDT Start.
1. Нажмите `Новый проект`.
1. Выберите тип проекта `Приложение 1С:Предприятия`.
1. Укажите название проекта.
1. В качестве расположения проекта EDT укажите каталог:

```text
<директория_репозитория>/itw_mis/src/itw_mis
```

1. Нажмите `Добавить проект`.
1. Если 1C:EDT Start предложит установить среду разработки или принять лицензионные соглашения, дождитесь установки и нажмите `Принять все`.
1. После завершения установки проект появится в списке в остановленном состоянии.

## 5. Настройка среды разработки

Эти настройки выполняются в 1C:EDT Start для установленной среды разработки и применяются ко всем проектам, которые ее используют.

Откройте настройки:

```text
1C:EDT Start > шестеренка > Среды разработки > нужная среда > шестеренка
```

Проверьте параметры:

* `Java VM` - путь к каталогу `bin` установленной Java. Для стандартной установки Axiom JDK 17 на Windows:

```text
C:\Program Files\Axiom\AxiomJDK-Pro-17-Full\bin
```

* `Максимальная выделенная память` - для больших конфигураций рекомендуется `8 Gb`.
* `Аргументы Java VM` - задайте короткий путь к временному каталогу, чтобы избежать проблем с длинными путями при импорте и экспорте:

```text
-Djava.io.tmpdir=C:\tmp
```

* `Язык интерфейса` - при необходимости выберите `Русский`.

## 6. Настройка антивируса

Для стабильной работы EDT добавьте в исключения антивируса или Windows Defender:

* приложение `1cedt.exe`;
* каталоги рабочих областей 1C:EDT;
* каталоги локальных Git-репозиториев.

## 7. Запуск проекта и привязка базы

1. В 1C:EDT Start запустите созданный проект.
1. Создайте пустую информационную базу.
1. В 1C:EDT создайте приложение и свяжите проект с информационной базой.
1. Настройте автономный сервер информационной базы, если он нужен для запуска и отладки.

## 8. Проверка перед началом работы

Перед разработкой проверьте:

* проект открывается из 1C:EDT Start;
* репозиторий склонирован по SSH;
* локальная ветка `dev` обновлена;
* рабочая ветка создана от `dev` и связана с issue;
* Git настроен по инструкции [git-windows.md](./git-windows.md);
* в 1C:EDT задана Java VM;
* для больших конфигураций выделено достаточно памяти;
* временный каталог EDT расположен близко к корню диска;
* информационная база создана и связана с приложением EDT.
